# Phase 2：Move 操作 + 工具输出增强

## 前置条件

Phase 1 完成：永久数字 ID 已就位，dependsOn 是 number[]，path 是缓存字段。

## Move 操作规格

### 工具参数

在 `roadmap_update` 中新增 `move_to` 参数：

```
roadmap_update(
  roadmapId: string,
  item_id: string,      // 要移动的项（path 或 #N 格式）
  move_to: string,      // 目标位置（path 格式）
)
```

### 支持的移动场景

| 场景 | item_id | move_to | 效果 |
|------|---------|---------|------|
| Task → 另一个 Story | "E2.S1.T3" | "E1.S9" | 从 E2.S1.tasks splice，push 到 E1.S9.tasks |
| Story → 另一个 Epic | "E2.S1" | "E1" | 从 E2.stories splice，push 到 E1.stories |

### 不支持

- Epic 移动（Epic 是顶层，没有更高级别）
- Task 移到 Epic 级别（层级不匹配，报错）
- Story 移到 Story 级别（层级不匹配，报错）
- 批量移动（后续迭代）

### 移动流程

```typescript
function moveItem(rm: RoadmapFile, itemId: string, targetPath: string): string {
  // 1. resolveItemId(itemId) 找到源项
  // 2. resolveItemId(targetPath) 找到目标容器
  // 3. 层级检查：Task 必须移到 Story，Story 必须移到 Epic
  // 4. 自身检查：不能移到自己所在位置
  // 5. 从源数组 splice 取出
  // 6. push 到目标数组
  // 7. rebuildPaths(rm) 重算所有 path
  // 8. 返回结果（新 path、源容器剩余数量）
}
```

### 移动后 dependsOn 不需要改

因为 dependsOn 存的是数字 ID（永久不变），移动只改变 path，不影响 ID。

### 返回格式

```
✅ 移动完成：
  "清理动态创建" (#42): E2.S1.T1 → E1.S9.T6
  
源 Story E2.S1 剩余 3 个 Task
```

## 工具输出增强

### 同时显示 path 和 #id

所有输出从：
```
E1.S9.T3: 实现撤销功能 [todo]
```
改为：
```
E1.S9.T3 (#42): 实现撤销功能 [todo]
```

### list 和 show 适配

- roadmap_list: 行末显示 #id
- roadmap_show: 每项标题后显示 #id
- roadmap_next: 建议中显示 #id
