# E6.S8: roadmap_plan 改为纯输出 + add 强制 planPath

## 背景

当前 `roadmap_plan` 直接写入 roadmap JSON，导致 AI 可以绕过 planPath 检查。实际工作流是：plan 是讨论沙盘，写入走 add，add 时强制要求计划文档。

## 改动清单

### T1: 提示词更新 — plan-description.md
- plan 定位改为"讨论沙盘推演，不写数据"
- 输出包含拆解结果 + 下一步 add 操作指南
- 移除关于 planPath 格式验证的描述（plan 不写数据了）

### T2: plan 输出格式定义 — plan-output-format.md（新建）
- 定义 plan 的输出模板
- 拆解结果部分：层级展示 Epic → Story → Task
- 操作指南部分：每个 Epic/Story 列出 write + add 命令，Task 列出 add_task 命令

### T3: tools-plan.ts 改造
- 去掉 `writeRoadmap` 调用，不再写入 JSON
- 去掉 `syncDoingChanges`、`getRoadmapFilePath` 等写入相关逻辑
- 新增 `formatPlanOutput(roadmap)` 函数：格式化输出拆解结果 + 操作指南
- 验证 JSON 格式（结构合法性），但不写入

### T4: add_epic/add_story 强制 planPath
- `addEpic`：没有 planPath → 拒绝，返回"请先用 write 创建计划文档"
- `addStory`：没有 planPath → 拒绝，返回"请先用 write 创建计划文档"
- `addTask`：不强制，给推荐提示即可

### T5: roadmap_plan 工具 description 更新
- 工具注册时的 description 反映新定位：讨论沙盘，输出拆解方案
- 不再提"create/update action"

### T6: 测试更新
- plan 测试：期望格式化文本输出，不期望写文件
- add_epic/add_story 测试：覆盖无 planPath 拒绝的分支
- 移除已过时的测试（如 sync、writeRoadmap 相关）

## 执行顺序

T1 → T2 → T3 → T4 → T5 → T6
