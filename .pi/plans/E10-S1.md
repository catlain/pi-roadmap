# Phase 1：永久 ID 数据模型 + 迁移 + 全部查找逻辑适配

## 目标

让 roadmap 的每一项（Epic/Story/Task）拥有一个永久的数字 ID，不随移动、重编号而变化。
dependsOn 改为引用数字 ID，彻底解决引用断裂问题。

## 改动清单（按顺序）

### Step 1: types.ts — 数据模型

```typescript
// 新增
interface RoadmapMeta {
  nextId: number;  // ID 分配计数器
  // ...原有字段
}

// 修改
interface Task {
  id: number;          // 永久 ID（新增，替代原来的 string）
  // path 不存储，从数组位置推导
  dependsOn?: number[]; // 数字 ID（替代 string[]）
  // ...其他不变
}
// Story、Epic 同理
```

### Step 3: store.ts — 迁移逻辑

readRoadmap 检测旧格式（id 是 string → 需要迁移）：

```typescript
function migrateIfLegacy(rm: any): RoadmapFile {
  if (typeof rm.epics?.[0]?.id === 'number') return rm; // 已是新格式
  
  let nextId = 1;
  const idMap = new Map<string, number>();
  
  for (const epic of rm.epics) {
    const epicNum = nextId++;
    idMap.set(epic.id, epicNum);
    epic.id = epicNum;
    
    for (const story of epic.stories) {
      const storyNum = nextId++;
      idMap.set(story.id, storyNum);
      story.id = storyNum;
      
      for (const task of story.tasks) {
        const taskNum = nextId++;
        idMap.set(task.id, taskNum);
        task.id = taskNum;
        if (task.dependsOn) {
          task.dependsOn = task.dependsOn.map(d => idMap.get(d) ?? d);
        }
      }
      if (story.dependsOn) {
        story.dependsOn = story.dependsOn.map(d => idMap.get(d) ?? d);
      }
    }
    if (epic.dependsOn) {
      epic.dependsOn = epic.dependsOn.map(d => idMap.get(d) ?? d);
    }
  }
  
  rm.meta.nextId = nextId;
  rebuildPaths(rm);
  return rm;
}
```

**关键**：迁移后立即写回文件（带 .bak 备份）。
